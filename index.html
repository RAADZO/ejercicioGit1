Introducción

JavaScript se ha convertido en uno de los lenguajes de programación más utilizados en la actualidad debido a la popularización en el uso de todo tipo de soluciones web para cualquier dispositivo electrónico. Este lenguaje nació como una herramienta para gestionar sencillas interacciones, por lo que en la especificación ECMAScript 2015 incorporó las implementaciones necesarias para resolver la complejidad de los algoritmos actuales.

En este módulo vamos a aprender de manera totalmente práctica las principales características de ECMAScript 2015, comenzando por el uso de la nueva sintaxis de declaración de variables y constantes, y cómo afecta a su ámbito.

Posteriormente, aprenderemos en profundidad cómo usar un conjunto de utilidades que son ampliamente empleadas por los desarrolladores JavaScript: template literals, chaining operator, nullish coalescence, proxies y las populares técnicas de desestructuring.

Y, para completar el aprendizaje de ECMAScript 2015, conoceremos en profundidad las nuevas funciones flecha, los métodos de alto nivel para arrays y las diferentes técnicas de manejo de asincronía, imprescindibles en el día a día del desarrollo de aplicaciones con JavaScript.

1. Nueva declaración de variables con “let” y “const”

Vamos a aprender de qué modo la aparición de ECMAScript 2015 ha modificado la forma de declarar variables en JavaScript, tanto sintácticamente como en relación con su contexto en el programa.

Comenzaremos por conocer qué es ECMAScript y cómo ha evolucionado hasta llegar a la especificación 6, que ha modificado sustancialmente la forma de programar en JavaScript.

1.1. Introducción a ECMAScript 2015

Vamos a detallar en qué consiste esta normalización del lenguaje JavaScript.

1.2. Declaración de variables con "let"

La declaración de variables con "let" permite definir variables con un ámbito de bloque, lo que mejora la gestión de memoria y evita errores comunes en el uso de variables globales.

1.3. Declaración de constantes con "const"

La declaración de constantes con "const" permite definir valores que no pueden ser reasignados, lo que ayuda a mantener la integridad de los datos a lo largo del código.

2. Template Literals

Los template literals permiten crear cadenas de texto de manera más sencilla y legible, incluyendo la posibilidad de interpolar variables y expresiones.

2.1. Sintaxis básica

La sintaxis básica de los template literals utiliza las comillas invertidas (``) y permite incluir variables utilizando la sintaxis ${variable}.

2.2. Multilínea

Los template literals permiten crear cadenas de texto multilínea sin necesidad de concatenar varias cadenas.

3. Funciones Flecha

Las funciones flecha proporcionan una sintaxis más corta para escribir funciones y no tienen su propio "this", lo que las hace ideales para callbacks y métodos de array.

3.1. Sintaxis básica

La sintaxis básica de una función flecha es (param1, param2) => { ... }.

3.2. Uso de "this"

Las funciones flecha no tienen su propio "this", por lo que heredan el "this" del contexto en el que fueron definidas.
